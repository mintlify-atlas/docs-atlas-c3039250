The PayphoneGo API returns JSON responses that are decoded into Swift types. This page documents the response structures and how they're handled in the iOS app.

## Response format

All API responses use JSON format with `Content-Type: application/json`. The API client automatically decodes responses into the appropriate Swift types.

## Response handling

The API client uses Swift's async/await pattern with automatic JSON decoding:

```swift
private func send<T: Decodable>(
    _ url: URL,
    method: Method = .get,
    headers: [String: String] = [:],
    body: (any Encodable)? = nil,
    as _: T.Type,
    expects statusCode: Int = 200
) async throws -> T
```

<Note>
  If the response body is empty, the decoder treats it as `null` to allow for nullable return types.
</Note>

## Status codes

The API client validates HTTP status codes and throws an error if the response doesn't match the expected status:

<ResponseField name="200" type="Success">
  Default expected status code for successful requests
</ResponseField>

<ResponseField name="Other" type="Error">
  Any status code other than the expected value results in an `API.Error.requestFailed` error
</ResponseField>

### Error response

When a request fails, the API client throws an error:

```swift
enum Error: Equatable, Swift.Error {
    case requestFailed(URLResponse)
}
```

**Error properties:**

<ResponseField name="requestFailed" type="Error case">
  Contains the `URLResponse` object for debugging and error handling
</ResponseField>

<ResponseField name="localizedDescription" type="String">
  Human-readable error message in the format: `"Request failed: {response}"`
</ResponseField>

## JSON decoding

The API client uses a default `JSONDecoder` with the following configuration:

```swift
JSONDecoder.default
```

**Key features:**

- Automatic snake_case to camelCase conversion via `@CodingKeys(.snake_case)`
- Type coercion for numeric types using `@CodedBy(ValueCoder<T>())`
- ISO 8601 date parsing for timestamp fields
- Nullable field support with Swift optionals

<Note>
  Empty response bodies are automatically converted to `null` to support nullable return types and prevent decoding errors.
</Note>

## Async/await usage

All API methods are asynchronous and must be called from an async context:

```swift
Task {
    do {
        let phones = try await api.phones()
        // Handle successful response
    } catch let error as API.Error {
        // Handle API error
        print(error.localizedDescription)
    } catch {
        // Handle other errors
    }
}
```

## Thread safety

The API client is marked with `@MainActor` to ensure all operations execute on the main thread, making it safe for use with SwiftUI views and state management.
