This guide will help you build and run the PayphoneGo iOS app on your device or simulator.

## Prerequisites

Before you begin, make sure you've completed the [installation](/installation) steps:

- Xcode 26.2 or later installed
- Project cloned and opened in Xcode
- Dependencies resolved via Swift Package Manager

## Open the project

<Steps>
  <Step title="Open PayphoneGo.xcodeproj">
    Navigate to the project directory and open the Xcode project:
    
    ```bash
    cd PayphoneGo
    open PayphoneGo.xcodeproj
    ```
  </Step>
  
  <Step title="Configure signing">
    1. Select the **PayphoneGo** target in the project navigator
    2. Go to the **Signing & Capabilities** tab
    3. Select your development team from the dropdown
    4. Xcode will automatically generate a provisioning profile
    
    <Warning>
    You may need to change the bundle identifier if `com.m1guelpf.PayphoneGo` is already in use.
    </Warning>
  </Step>
</Steps>

## Build and run

<Tabs>
  <Tab title="Simulator">
    <Steps>
      <Step title="Select a simulator">
        Choose an iOS simulator from the scheme selector (iPhone 16 Pro or later recommended)
      </Step>
      
      <Step title="Build and run">
        Press `Cmd+R` or click the Play button to build and run the app
      </Step>
      
      <Step title="Simulate location">
        The app requires location access. In the simulator:
        1. Go to **Features** → **Location** → **Custom Location**
        2. Enter coordinates (e.g., `37.7749, -122.4194` for San Francisco)
        
        <Info>
        You can also use predefined locations like "Apple" or "City Run" from the Features menu.
        </Info>
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Physical device">
    <Steps>
      <Step title="Connect your device">
        Connect your iPhone or iPad via USB and trust the computer if prompted
      </Step>
      
      <Step title="Select your device">
        Choose your device from the scheme selector in Xcode
      </Step>
      
      <Step title="Build and run">
        Press `Cmd+R` to build and install the app on your device
      </Step>
      
      <Step title="Trust the developer">
        On first install, you may need to trust the developer certificate:
        1. Go to **Settings** → **General** → **VPN & Device Management**
        2. Tap your developer certificate
        3. Tap **Trust**
      </Step>
    </Steps>
  </Tab>
</Tabs>

## First launch

When you first launch PayphoneGo:

<Steps>
  <Step title="Grant location permissions">
    The app will request location access. Tap **Allow While Using App** to enable location features.
    
    ```swift LocationManager.swift
    func requestAuthorization() {
        manager.requestWhenInUseAuthorization()
    }
    ```
  </Step>
  
  <Step title="View the map">
    The main screen displays an interactive map with payphone locations marked with teletype icons.
    
    - **Red markers**: Unclaimed payphones
    - **Gray markers**: Payphones with existing claims
  </Step>
  
  <Step title="Explore payphones">
    Tap any marker to view details about that payphone, including:
    - Address and location
    - Distance from your current location
    - Claim history (if any)
  </Step>
</Steps>

## Try the main features

### Navigate to the closest payphone

Tap the **Navigate to closest** button at the bottom of the map to find and navigate to the nearest unclaimed payphone:

```swift MainScreen.swift
func navigateToClosest() {
    guard let location = locationManager.location, let phones,
          let closestPhone = phones.min(by: { 
              $0.location.distance(from: location) < $1.location.distance(from: location) 
          }) else { return }
    
    closestPhone.navigate()
}
```

This will open Apple Maps with walking directions to the closest payphone.

### View payphone details

Tap any marker on the map to open a detail sheet showing:

- Full address and zip code
- Distance from your location
- Number of claims
- Claim history with timestamps and usernames

### Filter payphones

Use the filter button in the map controls to toggle between:
- **All**: Show all payphones regardless of claim status
- **Unclaimed**: Show only unclaimed payphones

<Note>
The filter UI is present in the code but currently displays all phones. This may be extended in future versions.
</Note>

## API integration

The app fetches payphone data from the PayphoneGo API on launch:

```swift MainScreen.swift
func fetchPhones() async {
    do {
        phones = try await api.phones()
    } catch {
        api.logger.error("Failed to get phones: \(error)")
    }
}
```

<Info>
Payphone data is cached locally using `TinyStorage` for offline access.
</Info>

## Debug features

In debug builds, you can access the network console:

<Steps>
  <Step title="Shake your device">
    Shake your physical device or press `Cmd+Ctrl+Z` in the simulator
  </Step>
  
  <Step title="View the console">
    The Pulse network debugger will appear, showing all API requests and responses
  </Step>
</Steps>

```swift App.swift
#if DEBUG
Atlantis.start(hostName: "supernova.local.")
NetworkLogger.enableProxy()
#endif
```

## Testing with location simulation

To test the app without moving around:

<Accordion title="Simulator location simulation">
  1. In Xcode, select **Debug** → **Simulate Location**
  2. Choose a preset location or create a custom GPX file
  3. The map will update with the simulated location
</Accordion>

<Accordion title="Custom GPX files">
  Create a GPX file with custom waypoints:
  
  ```xml
  <?xml version="1.0"?>
  <gpx version="1.1">
    <wpt lat="37.7749" lon="-122.4194">
      <name>San Francisco</name>
    </wpt>
  </gpx>
  ```
  
  Import it via **Debug** → **Simulate Location** → **Add GPX File to Project**
</Accordion>

## Common issues

<Accordion title="Location services not working">
  Make sure you've:
  - Granted location permissions when prompted
  - Enabled location services in device/simulator settings
  - Added the required `NSLocationWhenInUseUsageDescription` key to Info.plist
</Accordion>

<Accordion title="Build fails with signing error">
  1. Select your development team in the Signing & Capabilities tab
  2. Change the bundle identifier if needed
  3. Clean the build folder (`Cmd+Shift+K`)
  4. Try building again
</Accordion>

<Accordion title="No payphones showing on map">
  - Check your internet connection
  - Verify the API is responding at `https://walzr.com/payphone-go/api/phones`
  - Check the Xcode console for error messages
  - Try shaking the device to view the network console (debug builds only)
</Accordion>
